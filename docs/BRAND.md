# Rectly Brand Guidelines

## Purpose

Rectly exists to make perspective correction simple, private, and accessible to everyone.

The goal is to provide a tool that anyone can use immediately without installation, registration, or uploading images to a server.

---

## Vision

Become a trusted browser-based perspective correction tool by prioritizing simplicity, privacy, and usability.

---

## Design Principles

### Simplicity

The interface should be understandable without explanation whenever possible.

### Privacy

All image processing is performed locally in the browser.
User images are never uploaded or stored by the service.

### Browser First

Everything should work directly in a modern web browser without requiring installation or a server.

### Accessibility

Anyone should be able to use Rectly regardless of technical knowledge.

---

## UI Principles

* Minimize the number of steps required to complete a task.
* Keep frequently used functions visible.
* Avoid unnecessary dialogs and interruptions.
* Prefer consistency over visual complexity.
* Prioritize clarity over decoration.

---

## Brand Personality

Rectly should always feel:

* Simple
* Calm
* Reliable
* Trustworthy
* Professional

The product should never feel noisy, flashy, or overwhelming.

---

## Feature Policy

New features should satisfy all of the following:

* Work entirely in the browser.
* Preserve user privacy.
* Keep the interface intuitive.
* Improve the overall user experience.
* Avoid unnecessary complexity.

---

## Non-goals

Rectly is not intended to become:

* A cloud service
* An account-based service
* A photo editing suite
* A collaborative platform
* A server-dependent application

---

## Development Principle

When in doubt, prioritize improving the usability, reliability, and clarity of existing features over adding new ones.

Every feature should support Rectly's core values:
simplicity, privacy, and browser-first experience.
